C++ can have nasty syntax sometimes
Hmm today on IRC in the channel ##C++ @ Freenode someone posted the following code /*In g++ this does not work*/ template struct hello { hello(){}; ~hello(){}; template int world() { return _b+_a; } }; template void try_hello() { hello h; h.world(); } so i tried to help the person telling him that he should use a typename before h.world(); because it can ambitious for the compiler. It turns out that the right solution was h.typename world(); someone asked why is world() ambitious for the compiler and seldon said that the compiler can’t know the type of world in that context.
